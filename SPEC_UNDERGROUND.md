# 地底ステージ 仕様書（ぴよ氏の冒険 / Underground Stage）

**起案:** 2026-07-25（Ver.1.527 時点）
**位置づけ:** 単体の目玉リリース（1.4想定・大工程）。今日のアカバネ(1.527)＋HUD/ポーチ修正(1.526)は先に 1.3 として出す。
**正の仕様:** 本書。実装後の細部変更は HANDOFF/メモリの changelog に追記する。

## 0. ユーザー決定事項（2026-07-25）

- **構造 = R6=案山子 / R7=地底**（下記「1. 出現サイクル」で確定）。
- **地底のボス = 専用の大物ボス**（ラスボス格の新規ボス「闇の主」的なもの・案山子とは別）。案山子は通常寄りのボスとして温存。
- **時期 = 単体の目玉リリース**。まず本仕様書を固めてから実装に着手。
- 地底ステージは**ランダム生成でなく作り込み**（ハンドクラフト）でよい。
- 強制スクロールではない。ただし SMB 同様**左には戻れない**。
- ラウンド5終了後、スクロールが止まり土管に入るしかなくなる。落下したところから始まる。
- アカバネ（1.527・急降下雑魚）は「ボスが1周した以降」に顔ぶれを変える起点として調整する。

## 1. 出現サイクル（6ごと・R6=案山子ゲート → R7=地底）

エンドレスなので剰余で定義する。`gameRound` はボス撃破で +1 される既存仕様（gameplay.js:2780）をそのまま使う。

| 条件 | 内容 | 例 |
|---|---|---|
| `round % 6 === 0` | **案山子（門番ボス）** = 地底の直前ラウンド | R6, R12, R18… |
| `round % 6 === 1 && round >= 7` | **地底ステージ＋闇の主** | R7, R13, R19… |
| それ以外 | 通常ボスローテ `BOSS_KINDS`（rooster/hawk/egg/snake/owl） | R1–5, R8–11, R14–17… |

- 「6ごとに、案山子で門を開け→次のラウンドで地底に潜る」という**6ラウンド周期のミニアーク**。
- 案山子は通常ローテ配列には入れない（門番専用）。ローテのインデックスは「通常ボスのラウンドだけ進むカウンタ」で回す（実装フェーズで確定）。
- **要確認①**: この剰余定義（案山子=6の倍数 / 地底=6の倍数+1）でよいか。別案＝地底を初回R7のみ特別にし以降R13/R19…で案山子ゲート無しにする等。→ 現案は「毎回ゲート付き」で統一。

## 2. コアモデル =「拡張ボスアリーナ」（ここが設計の肝）

地底は**新しいゲームモードではなく、既存ボス戦の拡張**として実装する。これにより**ランキング不変条件を一切触らずに**成立する。

### 2.1 なぜ載るか（既存の裏取り）
- 距離は `gameState.distance = Math.floor(gameState.camera.x / 10)`（index.html:5405）だけで決まり、ランキングと `Lv = floor(距離/300)+1` は全部これ由来。**不可侵**（Firebase がルールで検証）。
- **ボス戦はすでにスクロールを止めている**：`setupBossArena` が `gameState.gameSpeed = 0` にする（gameplay.js）。`camera.x += gameSpeed`（index.html:5404）なので**ボス戦中は camera.x が進まない＝距離が凍る**。
- **ラウンドはボス撃破で `gameRound++`**（gameplay.js:2780・bossState.phase 5）。

→ つまり地底ステージ＝「1枚壁のアリーナ」を「作り込んだ複数画面のアリーナ」に拡張したもの。距離凍結もラウンド前進も**通常ボス戦と同じ挙動**。

### 2.2 距離・ランキングの扱い（確定）
- **地底の中では距離は凍結**（camera 前進ゼロ・`gameSpeed=0` 相当のサブステート）。
- 地底は**距離スコアに一切加算しない**。地上に戻ったら凍結前の距離から通常ランを再開。
- 結果：地底は「距離を稼ぐ区間」ではなく「作り込みチャレンジ＆報酬区間」。**ランキングは完全に無汚染**。この割り切りを受け入れる（ランキング保護 > 地底を距離に載せること）。
- ラウンド前進：**闇の主を撃破したら `gameRound++`**（通常ボスと同じ経路）。地上へ戻り、次の通常ボスは `bossDistanceFor(次round)` で先の距離にゲートされる（凍結中に距離が進んでいないので必ず前方＝矛盾なし）。

### 2.3 再利用する既存の仕組み
- `pipeRoomState` + `enterPipeRoom`（gameplay.js:437〜）＝**サブステートへのワープ／save/restore／出入り演出**のパターン。地底はこの「大型版」。
- `setupBossArena`＝スクロール凍結・壁・エンティティクリアの雛形。
- ボス撃破→`gameRound++`→通常地形の再開ポイント設定（`lastTerrainX`）の一連（gameplay.js phase5）。

## 3. 入場フロー（R7トリガー時）

1. R6の案山子を撃破 → `gameRound=7`。
2. 通常ランを `bossDistanceFor(7)` の距離まで進む。
3. その距離で**通常ボスの代わりに強制土管イベント**：スクロールを止め（`gameSpeed→0`）、画面手前に大きな縦土管を配置。ぴよ氏はそれ以上進めず、**土管に入る以外の選択肢がない**（左壁で戻れない）。ヒント表示「▼ もぐる」。
4. 土管に入る演出（既存のクリップ方式・穴に沈む）→ **暗転→地底へ**。
5. 地底の**天井の穴から落下してくる導入演出**（縦方向・無敵・入力ロック）。着地したら操作開始＝ここがR7の本体。

- **要確認②**: 強制土管の見た目。既存 `item_pipe.png` の拡大流用か、地底用に専用の大穴を描くか。

## 4. カメラ・操作

- **オートスクロール解除**：地底ではプレイヤー追従カメラ（`camera.x` はプレイヤー基準で算出）。
- **左壁クランプ（SMB式）**：`camera.x` は単調増加のみ。プレイヤーは到達済み最左より左へ戻れない（`player.x >= cameraLeftWall`）。
- 操作は既存のまま（左/右/ジャンプ/下スワイプ）。**追加操作は増やさない**（片手・デッドゾーン安全設計を維持）。
- **要確認③**: 追従カメラの縦方向。地底は縦にも掘る場合、縦追従を入れるか（実装コスト増）。初版は**横スクロール主体＋軽い縦**を推奨。

---

## 5. ステージ設計（作り込み・短く「降りて戦う」）

- **精密プラットフォーミングにしない**（1-4の溶岩ジャンプ地獄は片手操作と喧嘩する）。方向性は「降りて・進んで・戦う」。
- 長さ：**数画面（3〜5画面程度）**。奥に闇の主のアリーナ。
- 地形：作り込み固定レイアウト（レベルデータを持つ）。バネ/動く床/落ちる足場など既存ギミックは**控えめに**流用可。
- **反復対策**：地底は「ランドマーク」として同一 or 2〜3バリアントの使い回しを許容。**難度は敵ステータスのラウンドスケールで作る**（R7とR37で地形が同じでも敵が強い）。
  - **要確認④**: 初版はレイアウト1種で出すか、2〜3種を用意するか。
- **地底タイルセット**：暗い洞窟パレット。既存地形パレットのリカラー（`recolor` 機構＝アカバネで実装済の tint 方式が流用可）か、新規ドット。背景は BGM 込みで「地下」感を出す。

### レベルデータ形式（たたき台）
```
UNDERGROUND_LEVELS = [ {
  widthScreens: 4,
  platforms: [ {x,y,w,type}, ... ],   // 固定座標（ワールド原点=落下着地点）
  enemies:   [ {x,y,kind}, ... ],     // ラウンドでステータス倍率をかける
  hazards:   [ ... ],                 // 任意（トゲ/穴など・初版は最小）
  bossArenaX: <奥のボス部屋の開始x>
} ]
```

## 6. 敵

- 地底内の雑魚は**ラウンドでステータススケール**（速度/数）。既存の `1+(round-1)*0.3` 倍率の考え方を流用。
- 地底専用の見た目替え（洞窟コウモリ等）を tint リカラーで用意してもよい（アート軽い）。
- **アカバネ（1.527）の再調整**：現在 R11起点。「ボスが1周した以降」＝5ボス一巡＝**R6以降**が本来の意図。地底導入に合わせ、`DIVE_BIRD_ROUND` を見直す（地上の通常ランでの顔ぶれ変化の起点）。地底内に出すかは任意（初版は地上のみでよい）。

---

## 7. 闇の主（地底ボス）

- **ラスボス格の専用ボス**。通常ボス（rooster/hawk/egg/snake/owl）とは別枠・別サイズ・別演出。
- 既存ボスの `bossState` / phase / 撃破→`gameRound++` の枠組みは流用。攻撃パターンは新規。
- HP：通常ボスより高め（`BOSS_HP` 系の別枠 or 倍率）。ラウンドで微増。
- **アート工程**（[[piyo-sprite-motion-rule]] 厳守：立ち絵=OpenAI / 動き=Veo・1本ずつ確認）。地底ステージ本体の大工程の中で一緒に作る前提。
- **要確認⑤**: 「闇の主」のモチーフ（案山子より格上・地底の王）。案出しは実装着手時に別途ブレスト。

---

## 8. 退場・報酬

- 闇の主撃破 → `gameRound++`（=8）→ 撃破報酬（ゴールデンエッグ確定＋コイン等・**要確認⑥**で額を決める）。
- 地上へ復帰演出（土管から出る／せり上がる）→ 凍結していた距離から通常ランを再開。
- 死亡：地底内で死んだ場合の扱い＝通常のゲームオーバー（復活導線は既存のまま）。**要確認⑦**: 地底を「死なない部屋」にはしない（ボス戦だから）。土管ボーナス部屋（死なない）とは別物。

---

## 9. ランキング不変条件（厳守・再掲）

- 距離 `floor(camera.x/10)` / 速度 / `Lv=floor(距離/300)+1` に**一切触れない**。
- 地底は距離凍結＝スコアに寄与しない。Firebase の score 書き込みロジック・ルール検証は無改修。
- 検証は「地底の前後で距離が連続していること」「Lv計算が地底通過で飛ばないこと」を実測する。
